# 超市小票 OCR 识别与 Excel 导出工具 - 产品需求文档 (PRD) 

## 1. 产品概述 
### 1.1 产品背景
用户在日常生活中会积累大量超市购物小票，手动统计和记账费时费力。本产品旨在通过 AI 视觉大模型（OCR）技术，自动提取小票上的关键消费信息，并支持导出为结构化的 Excel 文件，极大提升个人记账和数据整理的效率。

### 1.2 产品定位
一个纯前端、纯静态、免费、安全（API Key 存本地）的个人效率工具。

### 1.3 目标用户
- 需要记录家庭开支的个人用户
- 需要整理报销凭证的职场人士
- 喜欢数字化管理生活数据的效率控

## 2. 核心功能需求

### 2.1 API Key 管理 (鉴权)
- **需求描述**：系统需要调用智谱 AI 的 GLM-4V-Flash 模型，需用户提供 API Key。
- **功能细节**：
  - 提供 API Key 输入弹窗/设置面板。
  - Key 仅保存在浏览器 `localStorage` 中，绝不上传任何第三方服务器。
  - 顶部导航栏提供直观的 Key 状态指示（未设置/已设置）。

### 2.2 图片上传与管理
- **需求描述**：支持用户方便地导入待识别的小票图片。
- **功能细节**：
  - 支持拖拽上传和点击选择文件。
  - 支持多选（批量上传）。
  - 上传后展示图片缩略图队列，支持移除不需要的图片。
  - 限制单张图片大小（建议 ≤ 5MB），支持常见格式 (JPG, PNG, WEBP)。

### 2.3 核心识别流程 (OCR)
- **需求描述**：调用大模型识别图片中的结构化数据。
- **功能细节**：
  - 提取字段：超市名称、购物日期时间、商品列表（商品名称、单价、数量、小计）、合计总金额。
  - 批量处理机制：多张图片排队逐一调用 API，避免并发超限。
  - 状态反馈：每张图片缩略图上显示当前状态（等待中、识别中、已完成、失败）。

### 2.4 数据校对与编辑
- **需求描述**：AI 识别可能存在误差，需提供人工校对手段。
- **功能细节**：
  - 采用“左图右表”的分栏设计（PC端），左侧预览原图，右侧展示识别出的表格数据。
  - 表格单元格支持直接点击编辑修改。
  - 自动重新计算：修改单价/数量时，不强制要求联动计算，但总金额应允许手动修正。

### 2.5 数据导出
- **需求描述**：将结构化数据导出为通用格式。
- **功能细节**：
  - 一键导出所有已识别完成的小票数据。
  - 导出格式为 `.xlsx` (Excel)。
  - 多张小票导出时，每张小票对应 Excel 中的一个独立 Sheet，Sheet 命名规则建议为“超市名-日期”。

## 3. 非功能需求

### 3.1 界面与交互 (UI/UX)
- **设计风格**：现代、极简、卡片式设计。
- **主题模式**：支持日间模式 (Light) 和夜间模式 (Dark) 无缝切换，保护视力。
- **响应式设计**：适配桌面端和移动端（移动端下“左图右表”降级为“上图下表”）。
- **反馈机制**：关键操作（如上传成功、识别成功、导出成功）需有 Toast 提示；耗时操作需有 Loading 动画。

### 3.2 性能与部署
- **架构**：Vite + React + TypeScript，纯前端静态应用。
- **部署方案**：通过 GitHub Actions 自动化部署至 GitHub Pages。
- **依赖限制**：不依赖任何后端数据库或服务端 API（除智谱 AI 接口外）。

## 4. 数据结构设计

### 4.1 核心业务对象
```typescript
interface ReceiptItem {
  name: string;       // 商品名称
  price: number;      // 单价
  quantity: number;   // 数量
  subtotal: number;   // 小计
}

interface ReceiptData {
  id: string;         // 唯一标识
  file: File;         // 原始图片文件
  previewUrl: string; // 本地预览URL
  status: 'pending' | 'processing' | 'success' | 'error';
  storeName?: string; // 识别出的超市名称
  date?: string;      // 购物日期时间
  items?: ReceiptItem[]; // 商品列表
  total?: number;     // 合计金额
}
```

## 5. 演进规划 (后续迭代考虑)
- **Phase 1 (MVP)**: 完成核心流程，跑通“上传->识别->导出”链路，部署上线。
- **Phase 2**: 增加本地历史记录保存（IndexedDB），下次打开网页还能看到之前的数据。
- **Phase 3**: 增加数据统计图表（如按月消费汇总、超市消费占比等）。
